# UX Flows

## Flow 1: First consultation prep

1. Lawyer taps **New Case**.
2. Adds client name/matter nickname.
3. Uploads documents or scans paper.
4. Records a 2-minute voice note: “Client says…”
5. App processes inputs.
6. App shows:
   - timeline draft;
   - people/entities;
   - facts known;
   - facts to verify;
   - missing documents;
   - questions to ask client.
7. Lawyer edits/accepts.
8. Lawyer exports or opens consultation brief.

## Flow 2: Court-day brief

1. Lawyer opens case in courthouse hallway.
2. Taps **Court-day brief**.
3. App shows:
   - next hearing/date/location;
   - procedural posture;
   - recent changes;
   - key facts;
   - unresolved issues;
   - documents to bring;
   - client reminders;
   - suggested talking points.
4. Lawyer can dictate a post-hearing note afterward.
5. App updates timeline and next actions.

## Flow 3: Appeal-prep packet

1. Lawyer creates appeal-prep workspace inside a case.
2. Uploads judgment/sentence/court docs.
3. Adds mitigation evidence.
4. App extracts:
   - procedural history;
   - sentence details;
   - grounds/issues candidates;
   - changed circumstances;
   - missing mitigation materials.
5. App creates an editable appeal-prep packet.

## Flow 4: US discovery triage

1. Lawyer uploads discovery batch.
2. App preflights pages/audio and estimated processing.
3. Lawyer approves cost/allowance use.
4. App indexes documents/transcripts.
5. Lawyer asks scoped questions:
   - “mentions of weapon”;
   - “timeline of arrest”;
   - “contradictions between report and witness statement”;
   - “all references to probable cause.”
6. App returns source-linked answer and evidence matrix.

## Flow 5: Large-processing cost guard

1. User selects huge upload.
2. App estimates:
   - document pages;
   - audio duration;
   - included allowance remaining;
   - estimated extra cost;
   - expected processing time.
3. User confirms or buys case pack.
4. Processing begins.

## UX principles

- Every output should have “why/source?” available.
- Important generated facts are editable.
- Use progressive disclosure: simple brief first, detail on tap.
- Voice capture must be extremely fast.
- Offline/local draft capture is valuable even if processing happens later.
